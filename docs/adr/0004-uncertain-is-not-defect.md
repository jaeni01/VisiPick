# 0004. '불확실'은 '불량'이 아니다 — 4분류와 반환 재검사

## 상태
Accepted (2026-05, 판정 로직 설계 시)

## 맥락
저신뢰 검출(신뢰도 < 0.40)이나 미검출을 어떻게 처리할지 정해야 했다.
가장 단순한 방법은 불량과 함께 폐기하는 것이지만, 순간의 각도·반사·블러로
정상품도 저신뢰가 나올 수 있다. 정상품을 한 번의 저신뢰 판정으로 버리면
과검(false call)이 수율을 갉아먹는다.

## 결정
판정을 4분류로 설계한다(`src/orchestrator/decision.py`):
- **PASS(NEEDED)** — 양품·레시피 필요 → 트레이 낙하
- **DUPLICATE** — 이미 수집한 종류 → Gate 반환 컨베이어
- **UNCERTAIN** — 저신뢰/미검출 → 폐기가 아니라 반환 컨베이어로 재투입, 재검사 기회 부여
- **REJECT(DEFECT)** — 파손·핀휨 확정 → 불량 게이트 격리

즉 "판단 보류"를 별도 등급으로 분리하고, 물리 동작은 중복과 같은 반환
경로를 태워 재검사를 유도한다.

## 검토한 대안
- **저신뢰 = 불량 폐기**: 구현은 단순하지만 정상품 폐기(과검)로 수율 손실.
- **신뢰도 임계값을 낮춰 전부 통과**: 미검(escape)이 늘어 불량이 트레이에 섞인다.
- **그 자리 재촬영**: 컨베이어를 세워야 해서 논스톱 인라인 검사 원칙과 충돌.

## 결과
- 과검과 미검을 동시에 억제한다 — 확실한 불량만 버리고, 애매한 것은 다시 본다.
- 트레이드오프 1: 재투입 순환으로 처리량이 줄고, 반복적으로 저신뢰가 나오는
  부품은 계속 순환할 수 있다(횟수 제한 미구현 — 확장 과제).
- 트레이드오프 2: 현재 WPF HMI 가 3분류 화면만 지원해, 송출 라벨 단계에서
  UNCERTAIN 을 DUPLICATE 로 합쳐 내보낸다(물리 동작은 동일한 Gate1 반환).
  내부 판정(verdict)은 4분류를 유지하므로 HMI 가 지원하면 매핑 한 줄로 되돌린다.
