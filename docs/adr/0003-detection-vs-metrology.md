# 0003. 분류는 딥러닝, 계측은 결정론적 CV 로 분리한다

## 상태
Accepted (2026-05, 비전 파이프라인 설계 시)

## 맥락
상부 카메라는 "이 부품이 무엇인가"(4종 분류 + 파손 검출)를 판단하고, 측면
카메라는 "핀이 기준에서 얼마나 벗어났는가"(핀 휨)를 판단해야 한다.
전자는 조명·각도 변화가 큰 혼류 컨베이어 환경의 분류 문제이고, 후자는
연속량(핀 개수, 간격 변동계수, 정렬 잔차)의 측정 문제다. 두 문제의 성격이
다른데 하나의 도구로 풀면 어느 한쪽이 약해진다.

## 결정
- **분류(detection)** — 상부: YOLOv8n 실시간 추론(`src/vision/classifier.py`).
  학습 기반이 혼류·조명 변화에 룰베이스보다 강건하다. (모델 학습·데이터셋은
  팀원 담당, 추론 통합·후처리는 본인)
- **계측(metrology)** — 측면: OpenCV 결정론적 기하 연산(`src/vision/pin_inspector.py`).
  핀 개수 → 누락·파손, 간격 변동계수(gap_cv) → 휨, polyfit 잔차 → 혼자 어긋난
  핀 검출. 같은 입력에 항상 같은 출력이 나오고 임계값의 물리적 의미가 명확하다.

## 검토한 대안
- **핀 휨도 DL 분류로**: "휨" 클래스를 학습시키는 방법. 휨의 정도를 연속량으로
  설명할 수 없고, 판정 근거를 임계값으로 제시하지 못해 재현성·설명가능성이 약하다.
- **전부 룰베이스 CV**: 분류까지 Hu 모멘트 등으로 푸는 방법(코드에 폴백으로 존재).
  혼류·조명 변화에서 오탐이 커 실환경 분류에는 부적합했다.

## 결과
- 문제 성격에 맞는 도구를 써서 분류는 강건하게, 계측은 재현 가능하고 설명
  가능하게 만들었다. 불량 코드도 물리량 기준으로 분리된다(개수=BROKEN, 분산=BENT_PIN).
- 트레이드오프: 파이프라인이 2계통이라 통합 판정 지점(`src/orchestrator/decision.py`)이
  별도로 필요하다. 또한 측면 계측은 표면 조명의 금속 반사 때문에 백라이트 도입
  전까지 참고(advisory) 수준으로 운용한다 — 계측 정확도의 상한은 알고리즘보다
  조명·광학이 먼저 결정한다는 한계를 명시해 둔다.
